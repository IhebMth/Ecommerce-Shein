# NOVA — Complete Backend Guide
### Separate repo · Node.js 24 · Supabase · Vercel · No SQLite

---

## WHY SEPARATE REPOS?

| nova-store (frontend) | nova-backend (backend) |
|---|---|
| Public GitHub repo | Private GitHub repo |
| Static files only | Node.js API + Admin panel |
| Deployed on Vercel | Deployed on separate Vercel project |
| yourstore.vercel.app | nova-backend.vercel.app/admin |
| If hacked → store goes down | Admin panel stays safe |
| No secrets, no DB access | All secrets live here only |

> The frontend calls the backend API via fetch().
> If someone finds a bug in your store, they cannot reach your admin panel
> because it is on a completely different domain and deployment.

## PHASE 0 — Setup

### What you do yourself:
1. Create a NEW GitHub repo called `nova-backend` — set it to **Private**
2. Clone it to your PC: `git clone https://github.com/yourname/nova-backend`
3. Node.js 24 is already installed — verify: `node --version` (should show v24.x.x)
4. Your Supabase, Upstash, Resend accounts are already set up from before
5. Create a NEW Vercel project linked to the `nova-backend` repo
6. Add all 9 environment variables to this new Vercel project (see below)
7. Add `.env.local` file locally with your real keys

### Environment variables (add to nova-backend Vercel project):
```
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_SERVICE_KEY=your-secret-key
UPSTASH_REDIS_REST_URL=https://your-db.upstash.io
UPSTASH_REDIS_REST_TOKEN=your-token
RESEND_API_KEY=re_your_key
RESEND_TO_EMAIL=your@email.com
ADMIN_SECRET=mX7$kP2@nQ9vR4wT6yU1zA3bC8dE5fG0

**What Claude builds:**

- `api/_lib/auth.js` — JWT cookie verifier, exports `verifyAdmin(req)`
- `api/_lib/rate-limit.js` — Upstash Redis limiter, exports `rateLimit()`

**Paste to Claude:**
```
Nova-backend. Node 24. No SQLite — Supabase everywhere.
Phase 1, Message 1.

Build 3 files in api/_lib/:

api/_lib/db.js:
  Uses @supabase/supabase-js
  Reads SUPABASE_URL and SUPABASE_SERVICE_KEY from env
  Export: getDB() — returns singleton Supabase client
  Auth: { persistSession: false }

api/_lib/auth.js:
  Export: function verifyAdmin(req)
  Read cookie "nova_admin_token" from req.headers.cookie
  Verify with jwt.verify() using process.env.JWT_SECRET
  Return decoded payload on success
  Throw error with status 401 if missing or invalid

api/_lib/rate-limit.js:
  Export: async function rateLimit(ip, key, max, windowSec)
  Use Upstash Redis REST API via fetch()
  Use UPSTASH_REDIS_REST_URL and UPSTASH_REDIS_REST_TOKEN
  INCR key, set expiry on first request
  Return { allowed: boolean, remaining: number }
  If Redis fails: allow request (fail open), log error

Return all 3 files.
```

### MSG 5-2 — supabase_rls.sql

**Paste to Claude:**
```
Nova-backend. Node 24.
Phase 5, Message 2.
Attach: database/schema.sql

Generate supabase_rls.sql with Row Level Security for all tables.
Use PostgreSQL syntax for Supabase.

For each table:
  ALTER TABLE x ENABLE ROW LEVEL SECURITY;

Policies:
  orders: SELECT service_role only | INSERT allow all
  products: SELECT allow all where active=true | write service_role only
  categories: SELECT allow all | write service_role only
